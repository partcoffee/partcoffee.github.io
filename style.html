1.0 - Typography

The primary fonts used are Roboto, Arial, and Montserrat. All with Helvetica Neue, Helvetica and sans-serif as fall-backs - in that order. A standard serif-font is also used as an addition to the main headline.

1.2 - Font weight

The fonts are using a font-weight of 300, which is classified as light for most fonts. This keeps the legibility high without sacrificing style throughout the project.

1.3 - Font usage

Roboto is used for headlines and sub-headlines for web - with a general line-height of 1 - 1.2.


Pixels are scale-independent.

Arial is a default font used for the body-text and throughout this page - with a line-height of 1.6.

Montserrat is a stronger font used for print headlines, navigational links, buttons and smaller body-text in all capital letters - with a general line-height of 1.2.

1.4 - Colour contrast and legibility

Text that is the same colour as the background is hard too read, but too much contrast makes reading uncomfortable. Any important text should maintain a minimum contrast ratio of 7.1:1 to pass the AAA contrast level of any sized text. Text of lesser importance can drop to a contrast ratio of 5.7:1.

As a reference, the body text colour holds a contrast-ratio of 7.5:1 to the background. (#555 to #FFF)

White text on transparent backgrounds (e.g. the navigation) holds a contrast ratio of 13.35:1 - 14.8:1 - depending on the background image underneath.

1.5 - Line length

The user will rarely visit the information-page more than once, but a good reading-experience should be a priority nonetheless.

The optimal line length for readability should consist of 50-75 characters. This particular page uses 70-80 characters per line, including spaces. This is arguably too much, but short paragraphs and repeated use of shorter lines makes up for it.

Important information should be bold and/or coloured to make the page easier to scan for the human eye.

2.0 - Colours

One Minute England is highly dependant on user content, which is why colours are few and far between. This highlights the content, and makes the UI, if at all present, a part of the background.

#222222 - dark background

#555555 - body text

#3fbcc5 - cosmetic elements

#de7373 - secondary elements

and other shades of black/white to indicate levels of importance.

3.0 - Animations

Animations are used to smooth out the user experience, and should be subtle as a result. The fading animations used on this site are linear 0.3 second animations. Easing is recommended for longer animations and/or animations involving movement.

A background colour of dark grey (#222) has been given to the body to avoid white flashes when the entire page is being faded out.

3.1 - Speed and responsiveness

To speed up the app, FastClick has been implemented on every page, including this one. This removes the 300ms delay between the physical tap and the firing of the click event on touch devices, making the website feel like a native application. This is recommended to keep the website quick and responsive on touch devices.

4.0 - Brand imagery

This project is about the user and the user content - thus our imagery is going to be limited. The only asset of importance is the logo, which should be used as is in white - assuming the background provides a contrast ratio of 11.3 or above. The contrast ratio is flexible and can be suited to the needs of the medium.

4.1 - Graphical elements

The project features buttons on every page to have a clear objective and achieve visual hierarchy with the titles.

The button(s) holds a 2dp solid border, creating a desired 'ghost button'. The button(s) adds 10% white (#FFF) to the background on hover, indicating a response to being moused over. The colour of the text and border should be inverted if the background is too light. The padding should be relative to the text within the button, but should keep touch-areas in mind.

Button Hover

There should also be a black bar with a height of 56dp on every-page to support the legibility of the navigation. This bar should hold a opacity of 0.1, but can and should be increased if needed.

4.2 - Icons

Apart from the logo, the project is using three font icons from the Entypo set from the Icomoon application. These are the three icons used - as an alternative to SVG-graphics.

5.0 - User content

The project is heavy on user content, but the user content should adhere to a certain quality. How strict we are will depend on the amount of content we receive, but the back-facing camera of an iOS device should be the lowest quality we accept. Audio can be included from the user content, but should flow with the music playing over the video itself.

6.0 - Accessibility

The recommended minimum touch target size for any on-screen element is 48x48 (px, dp). It's worth following this for ease of use, but it can be ignored for text links.

6.1 - Motion sickness

Users with vestibular disorders can become motion-disoriented, especially when trying to read text in front of a moving image, while users on the autism spectrum can be disturbed by rapid change. This is why the background image on the homepage is static.

6.2 - General annoyance

To avoid the general annoyance that comes with video and audio elements on the web, the user must interact with the video to play it. From there, mute-button and pause buttons are easily accessible, with the option to exit the video itself at any given time. This is expected behaviour on the web and should be a priority.
