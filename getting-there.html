WORKFLOW

This website - from start to finish

RSVP Yayy!

Tools

Firstly, thank you for your interest in how the site was made. We are constantly learning so this workflow might be dated within months, but we will do our best to keep this updated throughout the project.

This is a general list of tools worth mentioning as these are tools I am using every day for web development and deployment: Sublime Text, Emmet, GitHub, Sass, jQuery, Animate.css, and various jQuery plugins. I am also using Node JS (npm) when needed.

Early development

We started off with a six hour session on what the website would have to do, and how best to achieve that. Sketches were made, coffee was consumed and design discussions were had. Starting with a set plan instead of diving into the code straight away avoids bloated code that can be caused by repeated change in design.

I usually generate a new project from my simple, but functional framework. It's not as extensive as Foundation or Bootstrap, but it's enough to kickstart the project with a predefined grid system and a selection of Sass variables.

@mixin font($size) {
font-size: $size;
font-size: ($size / 16px) * 1rem;
}

@include font(36px);

Being able to define rem values with pixel values as a fall-back makes my life a lot easier. The support for rem is good, but when it fails, it's nice to have a bulletproof fall-back. I tend to follow this up by defining fonts and a colour scheme:

$headlines: 'Montserrat', sans-serif;
$bodytext: 'Helvetica Neue', Helvetica, Arial, sans-serif;

$titles: #383838;
$text: #555;
$teal: #36b3a8;
$teal-light: lighten(#36b3a8, 10%);

Defining fonts and colours like this makes the code easier to maintain, update and change - because if you change the colour once, it changes everywhere.

Illusions

We are using Animate.css to fade our content in - namely the entire splash screen and navigation. We also assigned a delay to the navigation to make sure everything faded in at a pleasant speed. This does not affect load times in a major way, but avoids the janky loading of most websites where elements pop into view at random.

The animations used on this website are 0.8 seconds, with two different delays defined. The site will wait 0.3 seconds before fading the content in, and another 0.2 seconds before the navigation fades in. The animation uses keyframes to change the opacity value form 0 to 1.

.animated {
animation-duration: .8s;
animation-fill-mode: both;
}

animation-delay: .3s;
animation-delay: .5s;

Manageable code

By importing documents into our main CSS document, we can seperate and manage our files easier. We are using Sass to do this, which means it will compile everything into one document, rather than just loading several documents. This saves us HTTP-requests, and makes sure we can serve a single minified CSS document for the entire page.

@import "map";

Writing and testing

I push changes to GitHub to test our changes on different devices, and I'm also running a simple Python HTTP server during my development. This makes a rather powerful workflow, and with Emmet, writing code is really quick.

That's about it - I hope this was insightful or interesting. Any questions or can be directed to email link in the footer.
